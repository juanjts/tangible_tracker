# Clean Code

## Objetivo

Generar código limpio, legible, mantenible y fácil de extender.

## Reglas

- Escribir código simple antes que código inteligente.
- Asignar una única responsabilidad a cada función y componente.
- Mantener funciones pequeñas y enfocadas.
- Utilizar nombres descriptivos y consistentes.
- Evitar duplicación de código (DRY).
- Priorizar la legibilidad sobre la optimización prematura.
- Extraer lógica repetida a funciones o servicios reutilizables.
- Mantener componentes React pequeños y reutilizables.
- Utilizar constantes y enumeraciones para valores fijos.
- Eliminar código muerto, comentarios innecesarios e imports sin uso.
- Mantener un estilo consistente en todo el proyecto.

## Nunca

- Crear funciones o componentes demasiado largos.
- Usar nombres genéricos como data, temp, item o value cuando exista un nombre más claro.
- Mezclar responsabilidades en una misma función.
- Repetir lógica existente.
- Comentar código para explicar una mala implementación.

## Checklist

- ¿El código es fácil de leer?
- ¿Cada función tiene una única responsabilidad?
- ¿Los nombres son descriptivos?
- ¿Existe código duplicado?
- ¿El cambio mejora la mantenibilidad del proyecto?
