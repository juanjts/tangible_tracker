# Review

## Objetivo

Revisar cada implementación antes de considerarla terminada, garantizando calidad, consistencia y mantenibilidad.

## Reglas

- Revisar el código generado antes de finalizar cada tarea.
- Verificar que se respete la arquitectura definida.
- Detectar posibles responsabilidades mezcladas.
- Identificar código duplicado.
- Buscar oportunidades de reutilización.
- Verificar nombres claros y consistentes.
- Detectar código innecesario o complejidad excesiva.
- Confirmar que la solución cumple los requisitos definidos.
- Proponer mejoras únicamente cuando aporten valor real.
- Explicar brevemente cualquier observación importante.

## Nunca

- Modificar el código durante la revisión sin aprobación.
- Cambiar la arquitectura del proyecto.
- Proponer sobreingeniería.
- Ignorar posibles problemas detectados.
- Dar una tarea por finalizada sin revisar el resultado.

## Checklist

- ¿Se respetó la arquitectura?
- ¿Existe código duplicado?
- ¿Las responsabilidades están bien separadas?
- ¿El código es limpio y fácil de mantener?
- ¿La implementación cumple completamente el requerimiento?
- ¿Existe alguna mejora opcional que valga la pena mencionar?
