# Development Workflow

## Objetivo

Desarrollar el proyecto de forma incremental, manteniendo la calidad del código y respetando la arquitectura definida.

## Reglas

- Analizar el requerimiento antes de escribir código.
- Explicar brevemente la solución propuesta antes de implementarla.
- Identificar los archivos que se crearán o modificarán.
- Implementar una sola funcionalidad o mejora por iteración.
- Mantener cada cambio pequeño, coherente y fácil de revisar.
- Respetar siempre la arquitectura y las reglas del proyecto.
- Reutilizar código existente antes de crear nuevas soluciones.
- Justificar cualquier nueva dependencia antes de incorporarla.
- Si una decisión cambia el diseño del proyecto, solicitar aprobación antes de implementarla.
- Mantener el contexto completo del proyecto durante todo el desarrollo.

## Nunca

- Generar grandes cantidades de código sin explicación.
- Modificar la arquitectura sin aprobación.
- Implementar funcionalidades fuera del alcance del proyecto.
- Crear código duplicado o innecesario.
- Asumir requisitos que no hayan sido definidos.

## Checklist

- ¿Se entendió el requerimiento?
- ¿La solución respeta la arquitectura?
- ¿El cambio es pequeño y fácil de revisar?
- ¿Se explicó la decisión antes de implementarla?
