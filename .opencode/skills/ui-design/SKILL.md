# UI Design

## Objetivo

Construir una interfaz moderna, limpia, consistente y responsive inspirada en aplicaciones SaaS.

## Reglas

- Utilizar Tailwind CSS para todos los estilos.
- Diseñar primero para escritorio, garantizando adaptación a móviles.
- Mantener espaciados, tamaños y tipografía consistentes.
- Utilizar cards con bordes redondeados y sombras suaves.
- Utilizar una paleta de colores neutra con un color principal para acciones.
- Mostrar estados hover, focus y disabled.
- Todos los elementos interactivos deben utilizar cursor pointer.
- Utilizar transiciones suaves cuando aporten a la experiencia.
- Mantener una jerarquía visual clara.
- Priorizar simplicidad y usabilidad sobre efectos visuales.

## Nunca

- Crear interfaces sobrecargadas.
- Mezclar estilos inconsistentes.
- Utilizar colores excesivos.
- Descuidar el diseño responsive.
- Romper la consistencia visual entre componentes.

## Checklist

- ¿La interfaz es responsive?
- ¿Los estilos son consistentes?
- ¿La experiencia de usuario es clara?
- ¿Se respetó el diseño SaaS?
- ¿Todos los elementos interactivos tienen feedback visual?
