# Git

## Objetivo

Mantener un historial de cambios limpio, incremental y fácil de entender.

## Reglas

- Sugerir commits pequeños y con una única responsabilidad.
- Utilizar mensajes siguiendo Conventional Commits.
- Agrupar cambios relacionados en un mismo commit.
- Proponer commits únicamente cuando una funcionalidad esté completa.
- Explicar brevemente qué incluiría el commit cuando sea necesario.

## Nunca

- Ejecutar comandos Git.
- Hacer git add.
- Hacer git commit.
- Hacer git push.
- Hacer git merge.
- Modificar el historial.
- Asumir que los cambios deben confirmarse automáticamente.

## Checklist

- ¿La funcionalidad está completa?
- ¿El cambio tiene una única responsabilidad?
- ¿El mensaje del commit es claro?
- ¿Solo se sugirió el commit sin ejecutar acciones?
